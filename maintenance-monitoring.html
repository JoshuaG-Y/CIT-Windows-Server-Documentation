Maintenance & Monitoring Plan

Routine Tasks for Server Upkeep and Performance Monitoring

1. Introduction & Purpose

This document outlines the routine maintenance and monitoring procedures for the CIT Windows Server 2025. Proactive maintenance and vigilant monitoring are essential for ensuring the server's stability, security, performance, and availability.

The goal is to prevent issues before they occur, detect problems early, and ensure the server continues to meet the needs of the CIT program.

2. Routine Maintenance Tasks

a. Daily Tasks:

  • Check Backup Status: Verify that all scheduled backups (OS, SQL, Application Data) completed successfully. Review backup logs for errors. (Refer to backup-recovery.html).
  • Review Key Event Logs: Briefly scan System, Application, and Security event logs for critical errors or warnings. Pay attention to logon failures, application crashes, disk errors.
  • Check Server Responsiveness: Basic check to ensure the server and key applications (e.g., hosted website) are responding.
  • Monitor Disk Space: Quick check on free disk space on all critical volumes (OS, Data, Logs, Backups).

b. Weekly Tasks:

  • Windows Updates: Check for and approve/schedule installation of pending Windows Updates. Reboot if necessary during a planned maintenance window.
  • Antivirus/Anti-malware Scan & Updates: Ensure Windows Defender (or other AV) definitions are up-to-date. Review scan logs. Consider a full scan if not scheduled automatically.
  • Review Performance Baselines: Compare current key performance metrics (CPU, Memory, Disk I/O, Network) against established baselines. Investigate significant deviations.
  • Check Application-Specific Logs: Review logs for IIS, SQL Server, and any critical hosted applications for errors or unusual activity.
  • Test User Account Access (Spot Check): Verify a test user account can log in and access necessary resources (if applicable).
  • Review Security Audit Logs: More in-depth review of security logs for suspicious activities, unauthorized access attempts, or policy violations.

c. Monthly Tasks:

  • SQL Server Maintenance:
    • Check database integrity (DBCC CHECKDB).
    • Update index statistics.
    • Reorganize/rebuild indexes as needed.
    • Review SQL Server error logs in detail.
  • Review and Archive Old Logs: Archive or clear older event logs and application logs as per retention policy to free up space and improve performance.
  • Disk Cleanup: Run Disk Cleanup utility or manually remove temporary files, old update files, etc.
  • Review User Accounts & Permissions: Audit user accounts, group memberships, and permissions. Remove or disable unused accounts. Ensure principle of least privilege is maintained.
  • Test Backup Restoration: Perform a test restore of a small, non-critical file or database to verify backup integrity (as per backup-recovery.html).
  • Review Server Documentation: Ensure server documentation (including this plan) is up-to-date with any changes.

d. As Needed / Ad-hoc Tasks:

  • Install software patches for third-party applications.
  • Apply security hotfixes.
  • Troubleshoot reported issues.
  • Perform hardware maintenance or upgrades (if applicable, with proper planning).
  • Respond to security incidents.

3. Monitoring

a. Key Areas to Monitor:

  • Performance:
    • CPU Utilization (% Processor Time)
    • Memory Usage (Available MBytes, Page Faults/sec)
    • Disk I/O (Avg. Disk Queue Length, Disk Read/Write Bytes/sec)
    • Network Interface (Bytes Total/sec, Output Queue Length)
  • Availability:
    • Server uptime (ping, RDP connectivity).
    • Service status (IIS, SQL Server, other critical services).
    • Website/Application responsiveness.
  • Security:
    • Logon attempts (successful and failed).
    • Firewall activity (blocked connections).
    • Antivirus alerts.
    • Changes to critical system files or user accounts.
  • Capacity:
    • Disk space usage on all volumes.
    • SQL Database size and growth.
    • Log file growth.
  • Backup Status: Success/failure of backup jobs.

b. Monitoring Tools:

  • Event Viewer: For System, Application, Security, Setup, and Forwarded Events, as well as application-specific logs (e.g., Windows Backup, IIS, SQL Server).
  • Performance Monitor (PerfMon): For real-time and historical performance data. Configure Data Collector Sets for baseline and trend analysis.
  • Resource Monitor: For detailed real-time view of CPU, memory, disk, and network usage by process.
  • Task Manager: For quick overview of running processes and performance.
  • Windows Server Backup Console: For backup job status.
  • SQL Server Management Studio (SSMS): For SQL Server performance (Activity Monitor, DMVs, Query Store), error logs, and Agent job status.
  • IIS Manager: For website status, application pool health, and request tracing.
  • PowerShell Cmdlets:
    • Get-EventLog, Get-WinEvent
    • Get-Counter (for performance counters)
    • Get-Service
    • Get-Disk, Get-Volume
    • Test-NetConnection
