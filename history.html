Chronology of Server Setup

This guide outlines the key stages and steps we took to set up the Windows Server 2025 environment. It's organized chronologically based on our process.

Phase 1: Operating System Installation

  1. Initial OS Setup:
    • Downloaded the Windows Server 2025 ISO image.
    • Created a bootable USB drive using a tool like Rufus or the Windows USB/DVD Download Tool.
    • Installed Windows Server 2025 (Desktop Experience) onto the designated CIT Hardware.
    • Encountered an issue with disk partitioning/formatting during installation; resolved by adjusting BIOS/UEFI settings and re-partitioning.
    • Installed necessary drivers for CIT hardware components post-installation via Device Manager or manufacturer downloads.

Phase 2: System Configuration & Basic Setup

  1. Date: [Date of system configuration]
  2. Actions:
    • Configured server hostname and joined to a workgroup/domain if applicable.
    • Set the system timezone: Using PowerShell Set-TimeZone -Id "Eastern Standard Time" or via Date and Time settings.
    • Configured a static IP address, subnet mask, default gateway, and DNS servers for the network adapter. This is crucial for the Virtual IP behind drhscit.org.
    • Installed essential roles and features via Server Manager (e.g., Web Server (IIS), .NET Framework).
    • Installed preferred utilities: e.g., KeePass, Git for Windows, Visual Studio Code, PowerShell 7.
    • Enabled Remote Desktop Protocol (RDP) for GUI access.
    • Installed and configured OpenSSH for Windows for secure shell access.
      • Added firewall rule for SSH (port 22).
      • Configured key-based authentication.
    • Performed initial Windows Updates.

Phase 3: Web & Application Server Setup

  1. Date: [Date of application server setup]
  2. Internet Information Services (IIS) Setup:
    • Installed IIS role with necessary components (e.g., ASP.NET, Application Development features).
    • Created a new website for the proof-of-concept application.
    • Configured website bindings (hostname for drhscit.org, IP, port 80/443).
    • Set the physical path to the application's root directory.
    • Configured an Application Pool for the application (e.g., .NET CLR version, identity).
  3. Application Deployment:
    • Cloned the full-stack web application source code from its Git repository to a designated folder on the server.
    • Installed application dependencies (e.g., using NuGet for .NET, npm for Node.js front-end build tools if applicable).
    • Configured application settings (e.g., web.config or appsettings.json) with database connection strings and other environment-specific values.

Phase 4: Database Setup (SQL Server Developer Edition)

  1. Date: [Date of database setup]
  2. Database System: SQL Server Developer Edition
  3. Installation: Downloaded and ran the SQL Server Developer Edition installer. Selected necessary features (Database Engine Services).
  4. Configuration:
    • Configured SQL Server instance (default or named).
    • Ensured SQL Server service was running.
    • Enabled TCP/IP protocol for remote connections using SQL Server Configuration Manager.
    • Configured Windows Firewall to allow traffic on the SQL Server port (default TCP 1433).
    • Created a new database for the application.
    • Created a SQL Server login/user for the application and granted necessary permissions to the database.
    • Configured database authentication (Mixed Mode if SQL logins are used).

Phase 5: Security Hardening & Best Practices

  1. Date: [Date of security configuration]
  2. Actions:
    • Applied security baselines (e.g., Microsoft Security Baselines).
    • Configured Windows Defender Antivirus and Firewall with appropriate rules and exclusions.
    • Removed or disabled unnecessary Windows features and services.
    • Implemented strong password policies.
    • Configured User Account Control (UAC).
    • Set up regular security audits and log reviews.
    • Enabled HTTPS for the web application using an SSL/TLS certificate (e.g., self-signed for testing, or a CA-issued certificate for drhscit.org).
